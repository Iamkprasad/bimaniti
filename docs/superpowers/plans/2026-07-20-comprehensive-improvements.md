# BimaNiti Comprehensive Improvements Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix bugs, clean up data issues, and improve code quality across the BimaNiti static site.

**Architecture:** Vanilla HTML/CSS/JS static site with a Node.js build script. Content lives in JSON files under `data/`. CI/CD via GitHub Actions.

**Tech Stack:** Vanilla HTML5/CSS3/JS, Node.js build script (esbuild for minification), GitHub Pages.

## Global Constraints

- Use vanilla JS — no frameworks or libraries
- Follow existing HTML/CSS naming conventions (BEM-like, custom properties)
- All changes must be backward-compatible with generated static files
- Don't remove or restructure files that are generated by the build script (post/*.html, learn/*.html, sitemap.xml, feed.xml)
- Author field stays as "BimaNiti" for all content entries
- Keep existing CSS custom property design system
